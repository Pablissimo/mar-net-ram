#language: pt-br

Negócio: Lista de Compras de Supermercado que a mãe passou

Metaforas: 
   Lista - É a lista do que a mãe quer que a gente compre
   Produto - É um item da lista de supermercado
   Sincronizar - É o ato de manter simultaneidade com todas as estações de trabalho 
   Replicar - Enviar suas modificações em tempo real para as demais estações de trabalho  
   
Funcionalidade: Cadastro de produtos
  Para manter a [[lista]] atualizada
  Eu, como usuário,
  Desejo poder adicionar, remover e alterar [[produtos]] da [[lista]]
  
	Cenario: Listar produtos existentes
	  Dado que eu quero acessar a [[Lista]]
	  Quando entrar no programa de Cadastro de Produto
	  Então a [[Lista] atual deverá ser mostrada
	  E um [Produto] sem nome deverá ser adicionado automaticamente ao final da lista para possibilitar [[Cadastrar um novo Produto]]
  
	Cenario: Cadastrar um novo [[Produto]]
	  Dado que eu quero cadastrar um <Produto>
	  Quando eu selecionar o <Produto> cujo nome está em branco
	  E digitar o nome do <Produto> que quero cadastrar
	  Então o <Produto> será automaticamente adicionado à [[Lista]]

	Cenario: Modificar um <Produto>
	  Dado que eu quero alterar o nome do <Produto>
	  Quando eu selecionar o <Produto>
	  E modificar seu nome 
	  Então a modificação no <Produto> será automaticamente gravada na [[Lista]]

	Cenario: Apagar um <Produto>
	  Dado que eu quero remover um <Produto>
	  Quando eu selecionar o <Produto>
	  E eu clicar no botão remover
	  E confirmar
	  Então o <Produto> será removido da [[Lista]]

Funcionalidade: Trabalhar off-line
    Para cada <operação> de algum <Produto> da <Lista>
	
	  Exemplos de <Lista>:
	    | Lista  | Produto |
	    | Almoço | Arroz   | 
	    | Almoço | Feijão  | 
	    | Almoço | Oleo    |	 	
	    | Lanche | Bolacha |
	    | Lanche | Leite   |
		
	  Exemplos de <operação>:
	    | operação  | Lista  | Produto(antes) | Produto(depois) |
	    | adicionar | Almoço |                | Macarrão        |
	    | apagar    | Almoço | Oleo           |                 |
	    | modificar | Almoço | Feijão         | Feijão Preto    |
	    	  
	Eu, como usuário,
	Desejo que algumas delas possam ser feitas, mesmo se não houver internet disponível
	  
	Cenario: off-line
	  Dado que estou sem conexão com a internet	  
	  Quando for feita alguma <operação> 
	  Então a <operação> ficará salvo localmente
	  E assim que a conexão se reestabelecer
	  E for [[sincronizado]] com o servidor, não haverá perda de dados
	 
    Cenario: on-line
	  Dado que estou conectado a internet	  
	  Quando for feita alguma <operação> 
	  Então deve-se [[sincronizar]] cada <operação> que fora registrada como pendente com o servidor
	  E, se houver <operação conflitante>, resolver conflitos para evitar problemas posteriores
	  E atualizar as informações que precisam ser mantidas no navegador para permitir acesso off-line
	  E [[sincronizar]] o <Produto> com o servidor

Funcionalidade: Edição com replicação
    Para cada <operação> feita em algum <Produto> da <Lista>
	Eu, como usuário,
	Desejo que minhas modificações sejam [[replicadas]] automaticamente para outros usuários
	E que modificações de outros usuários me sejam [[replicadas]]

    Cenario: Eu modifico a <Lista>
	  Dado que eu alterei um <Produto>
	  Quando a <operação> for concluida
	  Então as modificações deveram ser replicadas para todos os usuários que estão acessando essa lista
	  E a tela desses usuários devem ser atualizada

    Cenario: Outro usuário a <Lista>
	  Dado que outro usuário alterou um <Produto> da lista que estou acessando
	  Quando a <operação> for concluida
	  Então a modificação deverá ser replicada para minha tela
	  E assim por diante para todos as demais estações de trabalho

	  Esquema de cenário(Scenario Outlines) nos permite expressar de forma mais explicita exemplos atraves do uso de modelos com espaços reservados
	  	  
	  Esquema de cenário: Cadastro de produtos
	  Dado que tenho uma lista com os produtos a serem cadastrados
	  Quando eu acesso o sistema
	  E realizo todos os cadastros
	  Então os produtos estaram no sistema prontos para venda
	  
	  Exemplos:   
	  
	  Produto   | Quantidade   
      Arroz     |   200Kg    	 
	  Carne     |  1000Kg       
	  Farinha   |	 80Kg       
	  Ovos  	| 80 cartelas    
	  
	  Contexto(Backgrounds) nos permite adicionar um pouco de contexto aos cenários em um único recurso, um Background é um 
	  cenário sem tiluto que contem uma séries de passos, a diferença é na execução, onde a funcionalidade chama o 
	  Background que é executado antes de cada um dos seus cenários. 
		
	  Funcionalidade: Banco de Dados na nuvem
		 contexto:
			Dado o dono do Bando de Dados chamado Fulano
			E o Banco de dados se chama Banco do Fulano
			E um cliente chamado Beltrano		
			E uma empresa chamada META de propriedade do Beltrano  
	    
		Cenário: Fulano realiza backup do sistema da META
			Dado que esta estabelecendo conexao com o sistema
			E com o Banco do Fulano
			Quando a conexão for estabelecida
			E os dados forem copiados para o Banco do Fulano
			Então Fulano terá realizado o backup
